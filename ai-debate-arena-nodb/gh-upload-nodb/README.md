# Debate Arena — спор Gemini и Grok

Веб-приложение, где Gemini и Grok спорят друг с другом на заданную тему,
а табло считает слова и признаки согласия каждой стороны.

Стек: Next.js (App Router) на Vercel, Gemini API + xAI API вызываются
с сервера (ключи не светятся в браузере). Без базы данных — состояние
спора живёт в памяти браузера на время текущей сессии и не сохраняется
между визитами.

## Как это работает

1. На главной странице задаёшь тему и "личность" (system prompt) для
   каждого бота, выбираешь число раундов.
2. Боты по очереди отвечают друг другу — каждый ход дёргает соответствующий
   API (Gemini или xAI/Grok) через единственный серверный route
   `/api/debate/turn`, который принимает текущее состояние спора и
   возвращает обновлённое.
3. Если ключ xAI не задан, на балансе нет денег или запрос упал — приложение
   переходит в ручной режим и просит вставить ответ Grok вручную (например,
   скопировав его с grok.com), чтобы спор не подвисал.
4. Кнопка "новый спор" сбрасывает состояние и возвращает к форме.

## Локальный запуск

```bash
npm install
cp .env.example .env.local
# заполни .env.local своими ключами
npm run dev
```

Открой http://localhost:3000.

## Деплой на Vercel

### 1. Импортируй проект

На vercel.com → Add New → Project → выбери репозиторий. Framework Preset
определится как Next.js автоматически.

### 2. Задай переменные окружения

В Project Settings → Environment Variables добавь:

- `GEMINI_API_KEY` — ключ с https://aistudio.google.com/apikey
- `XAI_API_KEY` — ключ с https://console.x.ai (нужно привязать карту,
  бесплатного тира у xAI нет)
- `GEMINI_MODEL` (опционально) — по умолчанию `gemini-2.5-flash`
- `GROK_MODEL` (опционально) — по умолчанию `grok-4.3`

### 3. Задеплой

Если переменные были добавлены после первого деплоя — сделай Redeploy,
чтобы они подтянулись.

## Структура проекта

```
src/
  app/
    page.tsx                главная страница — форма + комната спора в одном useState
    api/debate/turn         единственный API route, stateless: принимает Debate, возвращает обновлённый
  components/                UI: карточка боя, реплики, табло, форма настройки
  lib/
    gemini.ts / grok.ts      клиенты моделей
    debate-engine.ts         построение промптов, подсчёт реплик
    analytics.ts              подсчёт слов и признаков согласия
    types.ts                  типы данных спора
```

## Известные ограничения

- Без сохранения: закрытие вкладки или обновление страницы стирает текущий
  спор. Шаринга по ссылке нет — это сознательный выбор для простоты.
- Аналитика "признаков согласия" — это простая эвристика по ключевым словам
  в тексте реплики, а не полноценный семантический анализ.
- Имена моделей (`GEMINI_MODEL`, `GROK_MODEL`) заданы через переменные
  окружения, а не зашиты в код — обе компании периодически переименовывают
  и заменяют модели.
